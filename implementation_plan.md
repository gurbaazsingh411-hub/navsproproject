# Referral Code System Implementation

Implement a strict referral code system where users must provide a valid referral code during signup, and admins can create new referral codes from the admin dashboard.

## User Review Required

> [!IMPORTANT]
> The referral code will become **strictly required** for all new signups. Users will not be able to create an account without a valid code that has been generated by the Admin.
> 
> Also, because we are creating a new table, **you will need to run a new SQL migration script** in your Supabase SQL Editor after I write it. Is this approach acceptable?

## Proposed Changes

### Supabase Database

#### [NEW] supabase/04_referral_codes.sql
- Create a `referral_codes` table to store valid codes.
- `code` column will be `TEXT UNIQUE NOT NULL`.
- Enable Row Level Security (RLS).
- Allow public read access so the signup page can validate codes before the user is authenticated.
- Allow authenticated users to insert codes (UI will restrict this to the admin).

### Frontend - Admin Dashboard

#### [MODIFY] src/pages/AdminDashboard.tsx
- Add a new "Create Referral Code" section.
- Allow the admin to type a custom referral code name and save it to the database.
- Fetch and display the list of all valid referral codes from the new table, alongside their usage statistics.

### Frontend - Signup Page

#### [MODIFY] src/pages/Signup.tsx
- Make the Referral Code input field required.
- Add a validation step in the `handleSignup` function: before creating the account in Firebase or Supabase, query the `referral_codes` table.
- If the code does not exist, display an "Invalid referral code" error and stop the signup process.

## Verification Plan
1. **Manual Verification**: 
   - Admin goes to the dashboard and creates a code (e.g., `SUMMER2026`).
   - A new user attempts to sign up with a random code (e.g., `FAKECODE`). They should see "Invalid referral code" and the account is not created.
   - The user attempts to sign up with `SUMMER2026`. The account is successfully created and tracked.
